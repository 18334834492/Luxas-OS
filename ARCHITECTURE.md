# 自研内核与深度桌面融合系统

## 系统架构

### 1. Kernel Lite (微内核)
- **职责**：IPC、线程调度、内存分页与共享、硬件中断转发
- **目标**：内核镜像 < 100 KB
- **特点**：无文件系统、无图形子系统、无驱动（全部用户态）

### 2. Luxas Shell (特权桌面服务)
- **职责**：合成器、窗口管理、开始菜单、任务栏
- **目标**：输入到渲染端到端延迟 < 1 ms
- **特点**：通过内核共享内存直接操作显存与输入事件

### 3. Windows 应用兼容层
- **职责**：Windows 应用运行支持
- **特点**：仅保留 25 个系统调用、DirectX → Vulkan 转译

### 4. 本地 AI 集成
- **职责**：AI推理引擎
- **特点**：开机即加载、侧边栏调用、标准接口

## 设计哲学

1. **边界清晰**：内核只做最小集合，其余全部推入用户态
2. **延迟优先**：共享内存 + 特权进程，消除不必要的上下文切换
3. **兼容即隔离**：Windows 兼容是子系统的事，内核对此无感知
4. **AI 原生**：AI 不是插件，是桌面的一部分，开机即可用

## 轻量性目标

| 指标 | 目标值 |
|------|--------|
| 开机内存占用 (完整桌面) | ≈ 280 MB |
| 冷启动到桌面 | ≤ 3 秒 |
| 完整系统体积 | 小于主流 Linux 发行版 |

## 技术路线

### Kernel Lite
- IPC 机制详细设计（同步/异步、能力传递）
- 线程调度器实现
- 内存分页与共享机制
- 中断转发框架

### Luxas Shell
- 合成器渲染管线 (Vulkan 后端)
- 窗口管理器实现
- 输入事件处理
- AI模块集成

### Windows 子系统
- 25 个系统调用的选取与实现
- Wine DLL 完整移植
- DirectX → Vulkan 转译层

### AI 集成
- 量化模型加载策略
- 模型内存布局优化
- 标准接口设计
